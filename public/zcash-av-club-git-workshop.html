Module 1 · 10 min

What is Git?

Git is a time machine for text files. GitHub is where you store that time machine in the cloud, so your whole team can use it.

💡
The core analogy
You know how Google Docs keeps a "Version History" of every edit? Git does that for entire folders of files — and it lets multiple people work on different parts at the same time, then merge their work together.

Three concepts you need to know

📦
Repository (repo)

A folder where Git tracks every change, forever. Think of it as a project binder that remembers every version of every page.

📸
Commit

A saved snapshot with a note explaining what changed. Like pressing "Save + Add a note" in a document. Each commit has an author and a timestamp.

🌿
Branch

A copy of the repo where you can experiment freely. The main branch is the "official" published version. Branches are for drafts and proposals.

🔀
Pull Request (PR)

A proposal to merge a branch into main. Others can comment, request changes, or approve. It's how the team reviews work before publishing.

What this means for the AV Club

Without GitWith Git
script_final_v3_REAL_USE_THIS.docx scripts/ep04-privacy-explainer.md (with full history)
Email threads with conflicting edits Pull Request with inline comments and approval
"Who changed the intro?" — no one knows Git log shows: @alice changed intro on March 15
Meeting notes lost in Telegram meeting-minutes/2026-03-27.md — searchable forever

A commit chain — what history looks like

a3f9d2 Add Q1 2026 meeting minutes
@mariana · 2 hours ago
7c12b0 Fix typo in episode 3 script intro
@leo · yesterday
d5e8a1 Draft script for "What is Zcash?" episode
@mariana · 3 days ago
2b34f7 Initial repo setup: README and folder structure
@leo · 1 week ago
✅
Good to know
In today's workshop we'll use GitHub's website — no command line, no installation. All the Git operations happen in your browser. If you later want to work locally, tools like GitHub Desktop make it easy.

Create the zcash-av-club repository

Module 4 · 15 min

Make Your First Real Commit

Now let's practice the core daily workflow: edit a file, write a good commit message, and see the change in history.

What makes a good commit message?

❌ BAD
updated stuff fix asdfasdf

Nobody knows what changed or why. Useless in history.

✅ GOOD
Add March 27 meeting minutes Fix typo in episode 4 script intro Draft "What is shielded tx?" script

Clear action verb + what changed. Future-you will thank you.

📝
The commit message formula
Verb + what + (optional) why. Examples: "Add notes from Q1 planning session", "Update script intro per team feedback", "Remove outdated resource links". Start with a capital letter, keep it under 72 characters, no period at the end.
D
Edit the README and commit the change
~7 min
  • On your repo homepage, click README.md, then the pencil icon ✏️ to edit it.
  • Replace the auto-generated content with a real description. Include: what the AV Club is, what this repo contains, and a link to the LCWG repo as inspiration (github.com/zcash/lcwg).
  • Use the "Preview" tab to see how the Markdown renders (# becomes a heading, **bold** becomes bold, etc.).
  • Commit with message: Write proper README with club description. Commit to main.
  • Go back to the repo home. Your README is live! Click the commit count link and find this commit in history. Click it to see the exact diff.

Module 5 · 10 min

Collaborate with Pull Requests

The full collaboration loop: propose a change on a branch, ask for review, discuss, and merge. This is how the AV Club reviews scripts before publishing.

The branch → PR → merge workflow

Main branch = published. Draft branch = work in progress. PR = request to merge draft → main.

🎬
For the AV Club specifically
Main branch = final approved scripts. A PR is like sending a draft script for review — team members comment on specific lines, you revise, and once everyone approves, the script gets merged to main (ready to record).
F
Create a draft script on a new branch
~10 min
  • In your repo, click "Add file" → "Create new file". Name it scripts/ep01-what-is-zcash.md.
  • Write a few lines of a draft script (it doesn't need to be complete — just a start). Use the template structure from scripts/template.md.
  • In the commit section at the bottom, select "Create a new branch" and name it draft/ep01-what-is-zcash. Click "Propose new file".
  • GitHub will redirect you to the Open Pull Request page. Add a description like: "First draft of the Zcash explainer script. Looking for feedback on the hook and tone." Click "Create pull request".
  • Share the PR link in the group. Have another participant open it and leave a comment on a specific line (click the Files changed tab → hover over a line → click the blue + button).
  • Once reviewed, click "Merge pull request" → "Confirm merge". The script is now on main! Check the scripts/ folder — it's there. Check the commit history — the merge is recorded.

⚡ Quick Reference

Git Cheat Sheet

Keep this open during your first few weeks of using GitHub. Everything you need for the AV Club workflow.

Core vocabulary

TermWhat it meansAV Club analogy
repository A folder tracked by Git with full history of all changes The AV Club's entire filing cabinet
commit A saved snapshot of changes with a message Saving a new version with a sticky note
branch An independent copy of the repo for drafting changes A photocopy you mark up before editing the original
main The default branch — the official published version The final, approved version of everything
pull request A proposal to merge a branch into main, open for review Sending a draft script for team review
merge Combining a branch's changes into another branch Incorporating approved edits into the final version
clone Downloading a copy of the repo to your computer Making a local copy of the filing cabinet
fork Your own copy of someone else's repo on GitHub Starting your own project based on someone else's template

The AV Club daily workflow

1
After each meeting: Create meeting-minutes/YYYY-MM-DD.md, fill in the template, commit directly to main.
2
Starting a new script: Create scripts/epXX-title.md on a new branch named draft/epXX-title. Open a PR immediately.
3
Reviewing a script: Open the PR → Files changed → click the blue + on any line to comment. Use "Request changes" or "Approve" buttons.
4
Script approved: Merge PR to main. Add a "published" label. Done!
